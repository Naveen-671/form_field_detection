# Operational Playbook

## Phase 1 — Local Development

### Prerequisites

- Python 3.10+
- NVIDIA NIM server running with Qwen3.5-397B-A17B (or accessible via network)
- Sufficient RAM for FFDNet-L + PaddleOCR (recommend 8GB+ available)

### Installation

```bash
cd field_detection

# Create virtual environment
python -m venv venv
venv\Scripts\activate    # Windows
# source venv/bin/activate  # Linux/Mac

# Install dependencies
pip install -r requirements.txt
```

### Environment Variables

Create a `.env` file or set these before starting:

```bash

NIM_API_URL=https://integrate.api.nvidia.com/v1/chat/completions
NIM_API_KEY=your-api-key-here
NIM_MODEL_NAME=qwen3.5-397b-a17b

# Optional (defaults shown)
DETECTOR_MODEL_PATH=FFDNet-L (1).pt

LLM_TIMEOUT_SECONDS=30
OCR_LANG=en
OCR_USE_GPU=false

```

### Startup Sequence

```bash
# Step 1: Ensure NVIDIA NIM is running (separate process/machine)
# Verify: curl http://localhost:8001/v1/models

# Step 2: Ensure FFDNet-L model file exists
# Place 'FFDNet-L (1).pt' in the project root directory
# Download from: https://huggingface.co/jbarrow/FFDNet-L

# Step 3: Start the application
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# Step 4: Verify startup
curl http://localhost:8000/health
# Expected: {"status":"ok"}

# Step 5: Open Swagger docs
# Browser: http://localhost:8000/docs
```

### Verify Model Loading

On startup, logs should show:
```
INFO detector_service: Loading FFDNet-L model from FFDNet-L (1).pt
INFO detector_service: FFDNet-L model loaded successfully
INFO ocr_service: Loading PaddleOCR engine
INFO ocr_service: PaddleOCR engine loaded successfully
```

If either fails, the application will still start but those endpoints will return errors.

## Incident Handling

| Scenario | Behavior | Operator Action |
|----------|----------|-----------------|
| LLM service unreachable | Auto-fallback to detector-only results | Check NIM server status, verify `NIM_API_URL` |
| LLM returns invalid JSON | Auto-retry once, then fallback | Review LLM logs, check system prompt |
| Detector fails to load | App starts but `/process-document` returns 500 | Check `DETECTOR_MODEL_PATH`, ensure `FFDNet-L (1).pt` exists in project root |
| OCR fails | LLM called without OCR context | Check PaddleOCR installation, check RAM |
| PDF corrupt/invalid | Returns 400 error | N/A (user error) |
| Out of memory | Process crash | Reduce image zoom or limit concurrent requests |

## Future Upgrades (Phase 2+)

- Fine-tune FFDNet-L on domain-specific document field detection dataset
- Replace PaddleOCR with faster/more accurate engine (e.g., EasyOCR, Tesseract 5)
- Replace LLM model with newer/smaller model when available
- Add embedding-based layout memory for cross-document learning
- Add Docker Compose for multi-service deployment
- Add GPU scheduling and batch inference
- Add response caching per page hash
